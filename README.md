# Nook

Nook is a free multilingual Discord bot for music, private voice channels, and clean server utilities.

You do not have to host anything to use it. The hosted Nook bot is free and ready to add to your server:

[Add Nook for free](https://discord.com/oauth2/authorize?client_id=1493567210948661330)

Need help, want to suggest a feature, or found something weird?

[Join the Nook Discord server](https://discord.gg/mf6993bmQ8)

Nook is also open source, so anyone who wants full control can host their own instance, customize playlists, change the code, and run it on their own infrastructure.

## What Nook Does

- Music playback through Lavalink.
- Search, autocomplete, queue controls, skip, join, and leave commands.
- Ready-made music packs through `/playlist`.
- Private voice channels with an interactive `/setup` panel.
- Server language configuration.
- English, French, German, and Spanish support.
- Persistent queue and private voice state with PostgreSQL and Prisma.
- Localized slash command names and descriptions.

## Music

Nook can play tracks from a title, URL, or autocomplete result. It keeps a persistent audio state, supports queue panels, and can offer curated playlists for people who do not know what to play next.

Main commands:

- `/play` - play a track or add it to the queue.
- `/playlist` - choose tracks from a ready-made playlist.
- `/queue` - show the current queue.
- `/skip` - skip the current track.
- `/join` - connect Nook to your voice channel.
- `/leave` - disconnect Nook from voice.

Playlist packs live in `src/services/audio/playlists.ts`. You can add tracks with Lavalink encoded data, title, author, duration, source, and artwork metadata.

## Private Voice Channels

Nook can create temporary private voice channels for your members. Admins can configure the system through `/setup`.

The setup flow lets you choose:

- The creator voice channel.
- The category for private rooms.
- The server language.
- Owner panel limits.
- Empty channel cleanup behavior.
- Temporary owner mentions.

## Languages

Nook supports four languages:

- English
- French
- German
- Spanish

The configured language is saved per server and reused across music responses, private voice panels, setup messages, and command localizations.

## Self Hosting

Self hosting is optional. Use it if you want to run your own bot token, database, Lavalink server, and custom playlist setup.

### Requirements

- Node.js 22 or newer
- Java 17 or newer for Lavalink
- PostgreSQL
- A Discord application and bot token
- npm

### Installation

Install dependencies:

```bash
npm install
```

Create a `.env` file:

```env
TOKEN=your_discord_bot_token
CLIENT_ID=your_discord_application_id
GUILD_ID=your_test_or_private_guild_id
DATABASE_URL=postgresql://user:password@localhost:5432/nook?schema=public

LAVALINK_HOST=localhost
LAVALINK_PORT=2333
LAVALINK_PASSWORD=youshallnotpass
LAVALINK_SECURE=false
```

Optional Spotify variables for LavaSrc and Spotify track URL metadata:

```env
SPOTIFY_CLIENT_ID=your_spotify_client_id
SPOTIFY_CLIENT_SECRET=your_spotify_client_secret
```

Optional Deezer API variables for `/play` playlist imports:

```env
DEEZER_ACCESS_TOKEN=your_deezer_access_token
```

Optional support server URL for the `/help` button:

```env
SUPPORT_SERVER_URL=https://discord.gg/your_invite
```

Generate Prisma and apply migrations:

```bash
npx prisma generate
npx prisma migrate deploy
```

Install Lavalink:

```bash
npm run setup:lavalink
```

Start Lavalink:

```bash
npm run start:lavalink
```

Build the bot:

```bash
npm run build
```

Start Nook:

```bash
npm start
```

## Project Structure

```text
src/                         TypeScript source code
src/commands/                Discord command entry points and thin orchestration
src/events/                  Discord event listeners
src/services/                Business logic and domain workflows
src/repositories/            Prisma and persistence access
src/types/                   Shared domain and DTO types
src/utils/                   Small pure helpers
src/config/                  Boot, environment, Discord builders, app wiring
src/domain/errors/           Explicit business and infrastructure errors
prisma/                      Prisma schema and database migrations
lavalink/application.yml     Lavalink server and plugin configuration
scripts/                     Lavalink setup and startup scripts
index.js                     Runtime entry point for compiled output
dist/                        Generated JavaScript output from TypeScript build
```

`src/` is where the TypeScript code is edited. `dist/` is generated by `npm run build`.

The codebase follows a strict separation:

- Commands and events translate Discord input into plain data and return Discord responses.
- Services contain business rules such as playback, queue votes, private voice permissions, and interaction workflows.
- Repositories are the only layer that talks to Prisma.
- `src/config/env.ts` is the only module that reads environment variables.
- Domain errors in `src/domain/errors/` are mapped to user-safe Discord messages by handlers.

## Lavalink

Nook uses Lavalink for audio playback. The repository includes `lavalink/application.yml`, with YouTube and LavaSrc plugin declarations.

These local runtime files are expected to stay outside the shared source:

- `lavalink/Lavalink.jar`
- `lavalink/logs/`
- `lavalink/plugins/`

They are downloaded, generated, or written while Lavalink runs.

## Discord Permissions

The hosted invite uses Administrator permission for a smooth setup. For a stricter custom invite, Nook needs permissions to:

- Send messages and use slash commands.
- Use embeds, links, and components.
- Connect and speak in voice channels.
- Manage channels for private voice creation.
- View audit logs for server join attribution.

## Customization

Good places to start:

- `src/services/audio/playlists.ts` for ready-made playlist packs.
- `lavalink/application.yml` for Lavalink sources and plugins.
- `src/config/env.ts` for typed runtime configuration.
- `src/services/privateVoice/` and `src/repositories/PrivateVoiceRepository.ts` for private voice behavior and persistence.
- `src/commands/` for command behavior and localizations.

## Troubleshooting

If slash commands do not appear, check `CLIENT_ID` and make sure Nook can register application commands.

If private guild commands are skipped, check `GUILD_ID`.

If music does not play, start Lavalink first and verify `LAVALINK_HOST`, `LAVALINK_PORT`, and `LAVALINK_PASSWORD`.

If Prisma fails, check that PostgreSQL is reachable and that `DATABASE_URL` points to the right database.
